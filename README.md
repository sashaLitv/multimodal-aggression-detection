## Про проєкт
Цей проєкт представляє собою інтелектуальну систему для виявлення агресивної поведінки за мультимодальними даними (аудіо та відео) на основі глибоких нейромереж. Архітектура системи побудована на незалежній обробці двох каналів даних із подальшим їхнім пізнім злиттям:

**Обробка відео**: для вилучення просторових координат людського скелета з кадру використовується модель YOLO11. Аналіз динаміки рухів та виявлення патернів фізичної агресії, здійснюється за допомогою комбінації графових згорткових мереж (GCN) та LSTM. 
Модуль використовує дані з наборів **NTU RGB+D 120** та **A dataset for automatic violence detection in videos**.  
Згідно з умовами використання зазначених наборів даних, надається офіційне підтвердження авторства (мовою оригіналу):
- (Portions of) the research in this paper used the NTU RGB+D 120 Action Recognition Dataset made available by the ROSE Lab at the Nanyang Technological University, Singapore.
- M. Bianculli, N. Falcionelli, P. Sernani, S. Tomassini, P. Contardo, M. Lombardi, A.F. Dragoni, A dataset for automatic violence detection in videos, Data in Brief 33 (2020). doi:10.1016/j.dib.2020.106587.

**Обробка аудіо**: застосовує згорткові нейронні мережі (CNN) для виділення акустичних ознак з мел-спектрограм та класифікації вербальної агресії. 
Модуль використовує дані з власного датасету, сформованого на базі аудіофрагментів українського телесеріалу «Школа» та власних записів.  

**Мультимодальне злиття**: для прийняття фінального рішення на основі даних з обох модальностей використовується перцептрон. 
Модуль використовує дані з власного датасету, що містить сцени фізичної, вербальної та змішаної агресії.

## Технологія функціонування
Загальний алгоритм роботи системи можна логічно поділити на чотири базові технологічні етапи.
1. Ініціалізація та захоплення даних. На цьому етапі визначається вхідна точка процесу, що виводить систему із стану очікування та активує механізм вхідного потоку даних.
2. Мультимодальний інтелектуальний аналіз. На цьому етапі система виконує паралельну обробку даних в двох інкапсульованих модулях: «Модуль виявлення вербальної агресії» та «Модуль виявлення фізичної агресії».
3. Агрегація та прийняття рішення. На цьому етапі виконується мультимодальне злиття, де розрізнені технічні показники від підсистем об’єднуються в інтегральну оцінку загрози для прийняття бізнес-рішення.
4. Фіналізація моніторингу та збереження даних. На цьому етапі забезпечується виведення інформації та збереження історії функціонування системи.

## Результати оцінювання
<div align=center>
  <img width="605" height="298" alt="Метрики оцінювання" src="https://github.com/user-attachments/assets/252fb701-a5e7-4bf1-8fe2-99bffcad4181" />
</div>
Під час навчання моделі було проведено тестування методом 5-фолдової крос-валідації. Для збалансування впливу класів та запобігання перенавчанню під час оптимізації функції втрат застосовувалися динамічні ваги, що усереднено дорівнюють 1.05 для класу нормальної поведінки та 0.97 для класу агресії. За результатами тестування розроблена система мультимодального злиття досягла загальної точності ~79%, а оцінка за метрикою F1-Score склала ~80%, що суттєво переважає ефективність ізольованих детекторів – аудіомодуля ~64.01% та відеомодуля ~50.09%. Разом з тим, використання алгоритму злиття забезпечило значне підвищення здатності системи фіксувати реальні загрози: підсумкова повнота виявлення зросла до ~83.08%, що демонструє абсолютну перевагу мета-класифікатора в порівнянні з обмеженими можливостями окремих модальностей, де аудіомодуль досягав повноти ~63.99%, а відеомодуль самостійно був здатен зафіксувати лише ~36.29% інцидентів.
Незважаючи на формально високі показники мультимодальної моделі злиття, до отриманих результатів слід ставитися з обґрунтованим науковим скептицизмом. Крім того, оскільки оцінка ефективності проводилася на обмеженій експериментальній вибірці, існує значний ризик недостатньої генералізації алгоритму.

## Розгортання системи
Система реалізована як автономний комплекс, що розгортається на єдиному пристрої – локальній робочій станції (ПК), що забезпечує закритий цикл обробки даних без потреби у зовнішніх серверах. В межах цієї станції функціонують два ізольовані середовища: нативне середовище Python для виконання аналітичних обчислень та платформа Docker для забезпечення роботи серверної частини. Взаємодія між середовищами та розміщення основних артефактів організовані наступним чином:
- середовище Python Runtime: містить артефакт ядра системи виявлення агресивної поведінки та артефакти ваг нейронних мереж (моделі YOLO, GCN-LSTM для відео та CNN для аудіо, перцептрону для злиття);
- середовище Docker Engine: містить контейнеризований артефакт бази даних на основі образу Azure SQL Edge, що дозволяє підтримувати стабільну роботу Microsoft SQL Server на будь-якій архітектурі, забезпечуючи збереження журналу інцидентів через локальний порт 1433.

## Поточні можливості та обмеження
Проєкт розроблено в рамках бакалаврської дипломної роботи як перевірку концепції мультимодального розпізнавання агресивної поведінки.  

Щоб забезпечити можливість запуску та роботи системи без використання високопродуктивних графічних процесорів (GPU), аналіз відео свідомо сфокусовано виключно на координатах скелетів, ігноруючи загальний візуальний контекст та задній фон. Хоча поточна архітектура демонструє хороші базові результати, точність системи може бути підвищена в майбутніх ітераціях шляхом:
- розширення та збагачення наборів даних, особливо для аудіомодуля, оскільки людські емоції та інтонації є вкрай різноманітними;
- додавання аналізу навколишнього середовища та об'єктів на фоні;
- вдосконалення паралельної роботи модулів, де при фіксації вербальної агресії відеомодуль міг би поза чергою обробляти підозрілий фрагмент;
- інтеграції системи з реальними камерами відеоспостереження для безперервного аналізу потоків даних.
